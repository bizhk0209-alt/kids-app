## プロジェクト概要

小学生・幼稚園児 3 人（はると・ゆき・ななみ）の学校行事・持ち物を管理する PWA。
Google Apps Script (GAS) が LINE/メール等から受信したお便りを OCR 処理し、JSON として返す。
フロントエンドはその JSON を取得して子供別にカード表示する。

---

## ファイル構成

```
kids-app/
├── index.html      # UI の骨格（タブ・検索・イベントリスト）
├── app.js          # ロジック全体（データ取得・レンダリング・チェックリスト）

├── manifest.json   # PWA マニフェスト
└── sw.js           # Service Worker（オフライン対応）
```

---

## アーキテクチャ

```
[GAS Web App (OCR処理 + データ返却)]
        ↑POST（今すぐ処理）
        ↓GET(?child=はると)
[app.js / フロントエンド]
        ↓
[localStorage（チェック状態永続化）]
```

### GAS エンドポイント

`app.js` の `CONFIG.GAS_URL` に設定済み（匿名アクセス可能な Web アプリとして公開）。

| メソッド | 用途 | レスポンス |
|---------|------|-----------|
| `GET ?child=<名前>` | イベント一覧取得 | `{ success: true, data: [...] }` |
| `POST` | OCR 処理トリガー | `{ success: true, message: "...", newFiles: N }` |

### イベントデータ構造（GAS から返却）

```json
{
  "日付": "2026-06-15",
  "行事名": "遠足",
  "場所": "○○公園",
  "持ち物": "お弁当、水筒、レジャーシート",
  "持ち物(ふりがな)": "おべんとう、すいとう、れじゃーしーと",
  "説明": "🕐 下校：14:30"
}
```

## 実装済み機能

| 機能 | 詳細 |
|------|------|
| 子供タブ切り替え | はると / ゆき / ななみ |
| イベントカード表示 | 日付・行事名・場所・持ち物プレビューをカード折り畳み状態で常時表示 |
| チェックリスト | カードを開くと持ち物チェックリスト表示。全チェックボタンあり |
| チェック状態永続化 | `localStorage` に保存（キー: `kidsapp_checks`） |
| ゆき専用ひらがな表示 | `needsKana: true` フラグ。漢字と読み仮名を 2 行同時表示（トグルなし） |
| 持ち物プレビュー | 折り畳み状態でも最大 4 件表示。5 件以上は「…他N件」表示 |
| 下校時刻バッジ | `説明` フィールドから `🕐 下校：HH:MM` を正規表現で抽出し表示 |
| 今すぐ処理ボタン | GAS に POST → 新規ファイルがあれば 2 秒後に再読み込み |
| 検索 | 行事名・場所・持ち物・ふりがな・説明を横断検索 |
| 日付ソート | 未来のイベントを先頭、過去を後尾に表示 |
| サンプルデータ | `GAS_URL` 未設定時はハードコードサンプルでプレビュー可能 |

## 子供ごとの設定（app.js: `CHILDREN`）

```js
const CHILDREN = {
  'はると': { cls: 'haruto', needsKana: false },
  'ゆき':   { cls: 'yuki',   needsKana: true  },  // ← ひらがな表示あり
  'ななみ': { cls: 'nanami', needsKana: false },
};
```

テーマカラーは `style.css` の CSS 変数で定義（`--haruto: #6dab84` など）。

## ローカル動作確認方法

GAS との接続なしでサンプルデータ確認する場合:

1. `app.js` の `GAS_URL` を空文字にする
2. `index.html` をブラウザで直接開く（または簡易サーバーで）

本番確認は GAS URL が設定済みのため、そのままブラウザで開けば動作する。
